Документація FastAPI

Огляд

Цей API надає можливості для реєстрації користувачів, входу в систему, перегляду списку всіх користувачів, а також очищення логів.

GET /

Головна сторінка API.

Відповідь: Ви зараз тут

GET /register/{username}/{password}

Реєстрація нового користувача.

Параметри:

  • username (стрічка): Ім'я користувача (повинно бути більше 5 символів і не містити пробілів).
  • password (стрічка): Пароль (повинен бути більше 5 символів і не містити пробілів).

Відповідь:

  • Успішна реєстрація: {"status": "success", "username": "ім'я користувача", "regist_date": "дата реєстрації"}
  • Помилка: 400 — "Пароль та юзернейм повинні бути більше 5 символів та не мати пробілів".
  • Помилка: 409 — "Даний юзернейм вже використовується".

GET /login/{user}/{password}

Вхід користувача до системи.

Параметри:

  • user (стрічка): Ім'я користувача (повинно бути більше 5 символів і не містити пробілів).
  • password (стрічка): Пароль (повинен бути більше 5 символів і не містити пробілів).

Відповідь:

  • Успішний вхід: {"status": "success"}
  • Помилка: 400 — "Пароль та юзернейм повинні бути більше 5 символів та не мати пробілів".
  • Помилка: 409 — "Даний юзернейм не використовується".
  • Помилка: 401 — "Пароль не вірний".

GET /alluser

Отримання списку всіх користувачів.

Відповідь:

            {
                "username1": {
                    "password": {
                        "salt": "some_salt",
                        "hash_password": "hashed_password"
                    },
                    "regist_date": "01-01-2025_12:00"
                },
                "username2": {
                    "password": {
                        "salt": "some_salt",
                        "hash_password": "hashed_password"
                    },
                    "regist_date": "01-01-2025_13:00"
                }
            }

GET /log

Отримання логів сервера.

Відповідь:

            2025-04-06 01:14:10,219 - uvicorn.access - INFO - 127.0.0.1:50366 - "GET /login/wsdf%7D/dd HTTP/1.1" 400
            2025-04-06 01:13:58,250 - uvicorn.access - INFO - 127.0.0.1:50363 - "GET /log HTTP/1.1" 200
            2025-04-06 01:13:54,532 - uvicorn.access - INFO - 127.0.0.1:50363 - "GET /register/wsdf%7D/dd HTTP/1.1" 400
            2025-04-06 01:13:36,617 - uvicorn.access - INFO - 127.0.0.1:50356 - "GET /log HTTP/1.1" 200

Користувач може очистити логи, натиснувши на кнопку "Очистити логи".

GET /log/clear

Очистити логи сервера.

Відповідь: HTML-сторінка з підтвердженням очищення логів.
